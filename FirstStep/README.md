## Задание 3 [#5] — Шаблон поста `post.php`

### Формулировка задания

> Готовимся отображать пост из БД. Сделаем шаблон для поста (файл post.php)  
> добавить код, который будет получать GET параметр с ключом postId и сохранять в переменную. Вывести значение в заголовке поста.  
> добавить массив данных для одного поста и заменить все статические данные поста на данные из массива (как делали с превьюшками постов).  
> p.s. В будущем, данные поста будем вытаскивать из БД по id, который получили из GET параметра postId

- Файл **`Maket/home/post.php`**: переменная `$postId` из `$_GET['postId']`, значение используется в `<title>` и рядом с заголовком поста, данные карточки берутся из массива `$post` (как у превью в `Maket/home/home.php`).

- В **`Maket/src/styles/home.css`** добавлены классы блока заголовка полной страницы поста (`.post-page__*`).

### Как открыть / показать

1. Запустите локальный сервер с PHP (например Laragon) и откройте каталог проекта как корень сайта **или** перейдите в папку макета.
2. Откройте в браузере:  
   **`http://localhost/.../Maket/home/post.php?postId=1`**  
   (путь зависит от того, как настроен виртуальный хост; главное — файл `Maket/home/post.php` и параметр `postId` в адресной строке.)

## Задание 4 [#10] — `api.php`: только POST, JSON и картинка в `static`

### Формулировка задания

> Создать файл api.php  
> этот файл должен обрабатывать запросы только методом POST.  
> Узнать, каким методом запросили скрипт: `$method = $_SERVER['REQUEST_METHOD'];`  
> Реализовать парсинг JSON запроса картинки и сохранить в файл в папку /static. Отправлять POST-запросы через postman

### Что сделано

Чтобы не смешивать с последующим заданием (создание поста в БД через multipart), логика **JSON + base64/data URL → папка `static`** вынесена в отдельный файл:

- **`api_base64_to_static.php`** — только **POST**, проверка `REQUEST_METHOD`, разбор JSON, поле **`image`** (base64 или data URL), сохранение файла в **`static/`**, ответ JSON.

Исходное задание с путём **`/static`** закрыто этим скриптом; папка **`static/`** есть в проекте (с `.gitkeep`).

### Как открыть / показать (Postman)

1. Метод **POST**, URL: **`http://localhost/.../api_base64_to_static.php`**
2. **Body** → **raw** → **JSON**, пример:

```json
{
  "image": "data:image/png;base64,iVBORw0KGgoAAAANSUhEUgAAAAEAAAABCAYAAAAfFcSJAAAADUlEQVR42mP8z8BQDwAEhQGAhKmMIQAAAABJRU5ErkJggg==",
  "filename": "demo.png"
}
```

3. В ответе будет путь вида **`static/demo.png`**. Файл появится в папке **`c:\WEB\static\`**.

4. Для проверки **405**: отправьте тот же URL методом **GET** — вернётся сообщение о том, что разрешён только POST.

## Задание [#10] — База `blog`, таблицы `user` и `post`, SQL-файлы

### Формулировка задания

> Создайте базу данных blog  
> Завести в базе данных табличку post, содержащую информацию о посте (заранее продумать поля) и user (информация о пользователях)  
> SQL-запрос для создания таблички сложить в data/sql/table  
> Заготовить данные о постах в виде sql-файлов с конструкцией INSERT INTO, где прописать вставку информации о каждом из постов(см. блок примеры), SQL-файл с информацией о статьях сложить в data/sql/posts

### Что сделано

- **`data/sql/table/create_tables.sql`** — `CREATE DATABASE blog`, таблицы **`user`** и **`post`** (включая поле **`subtitle`** у поста, внешний ключ на пользователя).
- **`data/sql/table/insert_users.sql`** — два тестовых пользователя.
- **`data/sql/posts/insert_post_1.sql`**, **`insert_post_2.sql`** — отдельные вставки постов (как в примерах макета / короткий второй пост).
- **`data/sql/posts/insert_posts.sql`** — **одним файлом**: пользователи + два поста («Прогулка по зимнему городу», «Мой идеальный завтрак»), чтобы можно было сдать монолитный вариант, как в задании с «примерами».

**Временно удаляли** объединённый `insert_posts.sql` при разбивке на файлы — по просьбе он **восстановлен**; разбитые файлы **не удалялись**.

### Как открыть / показать

1. Откройте **phpMyAdmin** или клиент MySQL.
2. Выполните по порядку:
   - `data/sql/table/create_tables.sql`
   - `data/sql/table/insert_users.sql`
   - `data/sql/posts/insert_post_1.sql` и `insert_post_2.sql`  
   **или** для одного прогона сидов пользователей и статей: только **`insert_posts.sql`** (если база пустая; если пользователи уже есть — см. комментарий в начале `insert_posts.sql`).

## Задание [#20] — Чтение постов из MySQL в PHP

### Формулировка задания

> Реализовать чтение данных о постах из базы данных MySQL в PHP-скриптах на бекенде

### Что сделано

- **`config/database.php`** — подключение PDO к БД **`blog`**, убран отладочный вывод; при вызовах API с флагом **`BLOG_JSON_API`** ошибка БД отдаётся JSON.
- **`index.php`** — выборка опубликованных постов с **`JOIN`** на **`user`**, поля **`title`**, **`subtitle`**, текст для ленты (excerpt или обрезка **`content`**), относительная дата, ссылка «ещё» на **`post.php?postId=...`**.
- **`post.php`** (в **корне** `c:\WEB`) — один пост по **`postId`**, только **`published`**, иначе **404**.
- **`includes/published_relative.php`** — формат даты («N минут назад», «вчера», дата).
- **`src/styles/main.css`**, **`src/styles/home.css`** — стили под разметку ленты и полного поста в корне проекта.

### Как открыть / показать

1. Убедитесь, что БД заполнена (см. раздел SQL выше).
2. В браузере: **`http://localhost/.../index.php`** — лента с постами из БД.  
3. Перейдите по «ещё» или откройте **`post.php?postId=1`** (подставьте существующий `id`).

## Задание [#20] — Админка: сохранение поста в БД через POST (JSON + файл картинки)

### Формулировка задания

> [Готовимся к админке] Сохранять данные о посте в БД через POST запрос  
> Создать файл api.php - этот файл должен обрабатывать запросы только методом POST  
> Узнать, каким методом запросили скрипт: `$method = $_SERVER['REQUEST_METHOD'];`  
> Реализовать парсинг JSON запроса + картинки  
> Обработать новые параметры, если данные некорректные - сообщить пользователю  
> Картинку сохранить в папку images  
> Сформировать запрос к БД, выполнить SQL запрос  
> Отправлять POST-запросы через postman  
> Сформировать руками JSON (потом JSON будет формироваться на клиенте через JavaScript)  
> p.s. картинку принимаем на backend файлом  
> Проверить на странице home появился ли у вас новый пост

### Что сделано

- **`api.php`** — только **POST**; **`multipart/form-data`**: текстовое поле **`json`** или **`data`** (строка JSON с полями поста) + файл **`image`**; валидация; сохранение файла в **`images/`**; **`INSERT INTO post`**; ответ JSON с **`post_id`**, **`slug`**, **`featured_image`**.
- Папка **`images/`** (с `.gitkeep`).
- В начале **`api.php`** в комментарии указано, что сценарий **JSON+base64 в static** — в **`api_base64_to_static.php`**.

### Пример JSON для поля `json` в Postman (двойные кавычки)

```json
{
  "title": "Post number 13",
  "subtitle": "Черновик из API",
  "content": "Полный текст поста для проверки.",
  "excerpt": "Кратко для ленты.",
  "user_id": 1,
  "slug": "post-number-13",
  "status": "published",
  "views": 0
}
```

### Как открыть / показать

1. Postman: **POST** → **`http://localhost/.../api.php`**
2. **Body** → **form-data**:
   - **`json`** (Text) — строка JSON выше;
   - **`image`** (File) — любой поддерживаемый файл (JPEG/PNG/GIF/WebP).
3. В ответе **`"ok": true`** и **`post_id`**.
4. Откройте **`index.php`** — новый пост с **`status": "published"`** должен появиться в ленте (вверху списка по дате публикации).

**Важно:** в таблице **`user`** должен существовать **`user_id`** (например `1` после сидов).
